SonarCloud Configuration

Connect your SonarCloud account to pull project findings.

Generate a token at SonarCloud → My Account → Security.

Required to list SonarCloud projects. Find it in your SonarCloud organization URL or organization settings.

Socket.dev Configuration

Connect your Socket.dev organization to pull dependency risk findings.

Generate an organization token at Socket.dev → Organization Settings. Required scope: alerts:list. No repository write scopes are required.

Required to scope Socket.dev alerts to the current repository during sync. Find it in your Socket.dev dashboard URL: https://socket.dev/dashboard/org/<org-slug> (for example, xxx-xxxxxx).

Snyk Configuration

Connect your Snyk account to import project vulnerabilities.

Create a personal or service account token at Snyk → Account Settings.

Required to scope Snyk issues to the current repository during sync. Find it in your Snyk URL (https://app.snyk.io/org/<org-id>) or under Snyk → Organization Settings.

Semgrep Configuration

Connect your Semgrep deployment to pull code and supply-chain findings.

Generate a token at Semgrep → Settings → Tokens.

Required to scope Semgrep findings during sync. Open your Semgrep dashboard and copy the URL segment after /orgs/; for example, https://semgrep.dev/orgs/example-org/projects/scanning uses example-org.

Finding Types

Semgrep serves Supply Chain findings from a separate API mode, so selecting both writes two scanner sources that share the same token and deployment.
